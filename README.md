# LlamaIndex Examples

This repository contains small, practical examples built while following the Real Python tutorial:

- [LlamaIndex Examples (Real Python)](https://realpython.com/llamaindex-examples/)

## Credits

- Tutorial and guidance: Real Python article linked above.
- Core libraries used here: `llama-index` and `python-decouple`.

## Prerequisites

- Python `>=3.10`
- [`uv`](https://docs.astral.sh/uv/)
- An OpenAI API key

## Setup

1. Install dependencies:

```bash
uv sync
```

2. Create a `.env` file in the project root:

```env
OPENAI_API_KEY="your_api_key_here"
```

## Project Files

- `main.py`: Basic one-shot example that loads `data/pep8.rst`, builds an in-memory index, and runs a single query.
- `src/single_query.py`: Persists the index to `storage/` and reuses it on next runs; executes one query.
- `src/async_query.py`: Same persisted index flow, but runs multiple queries asynchronously with `asyncio`.
- `src/llm_integration_query.py`: Uses a custom OpenAI LLM (`OpenAI(model="gpt-5.1")`) in the query engine.
- `data/pep8.rst`: Input document used to build the index.
- `storage/`: Generated folder containing persisted index artifacts.
- `pyproject.toml`: Project metadata and dependencies.
- `uv.lock`: Lockfile generated by `uv` for reproducible environments.
- `.env`: Local environment variables (API key). Keep this file private.

## How to Run (with uv)

From the project root, run:

```bash
uv run main.py
uv run src/single_query.py
uv run src/async_query.py
uv run src/llm_integration_query.py
```

## Notes

- The first run of scripts that use persistence (`src/single_query.py`, `src/async_query.py`, `src/llm_integration_query.py`) creates `storage/`.
- Later runs load the index from `storage/` for faster startup.
- If you want to rebuild the index from scratch, remove `storage/` and run again.
